# Validation record

Status: **ready to share with caveats**.

## Checks passed

- complete coverage of 3 shifts × 5 seeds × 4 ranks for LoRA and DoRA;
- no duplicate `(scenario, seed, method, rank)` rows;
- accuracy and macro-F1 values are within `[0, 1]`;
- learning-rate and early-stopping selection use target validation, not target test;
- paired DoRA−LoRA deltas use identical seed, shift, rank, source split, and test corruption;
- trainable parameter counts verified by unit tests;
- rank-4 additive positive control reaches numerical zero for both LoRA and DoRA;
- headline numbers are generated from saved raw result CSVs by `analyze_results.py`;
- every poster figure is generated by `make_figures.py` from saved result tables.
- notebook cells were validated top-to-bottom and outputs embedded by the documented in-process fallback because the hosted workspace blocks Jupyter kernel sockets; the same notebook can be rerun normally in a local Jupyter environment.

## Required caveats

- The synthetic comparison measures representational capacity. LoRA receives its exact SVD optimum; DoRA receives a feasible solution using the known generative decomposition.
- The real-data benchmark is a small image proxy, not an LLM reproduction.
- Five paired adaptation seeds share one fixed pretrained backbone.
- Intervals are 95% paired t-intervals and are necessarily coarse at `n=5`.
- The active target adapter can reduce clean-domain accuracy; the frozen base is still recoverable by disabling the adapter.
